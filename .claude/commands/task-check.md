# タスク進捗チェック

## 指示

`.kiro/specs/chronolog/tasks.md` の仕様書を読み込み、タスク番号 **$ARGUMENTS** の実装状況を確認してください。

## 確認手順

1. `.kiro/specs/chronolog/tasks.md` を読み込み、指定されたタスク番号のサブタスクと要件を把握する
2. プロジェクトのソースコード（`src/`配下）を調査し、各サブタスクが実装済みかどうかを判定する
3. 以下のフォーマットで結果を報告する

## 出力フォーマット

```
## タスク [番号]: [タスク名]

### サブタスク別チェック結果

| サブタスク | 状態 | 備考 |
|-----------|------|------|
| [サブタスク内容] | ✅ 完了 / ❌ 未実装 / ⚠️ 一部実装 | [詳細] |

### 総合判定
- 完了: X / 全Y サブタスク
- 判定: ✅ タスク完了 / ❌ 未完了

### 未実装項目の詳細（未完了の場合）
- [未実装のサブタスクと、何が不足しているかの説明]
```

## 注意事項

- `*` マークが付いたサブタスクはオプションであることを明記する
- 実際のソースコードを確認して判定すること（ファイルの存在だけでなく、中身も確認する）
- テスト関連のサブタスクはテストファイルの存在とテストケースの内容を確認する
- 判定に迷う場合は ⚠️ として理由を備考に記載する
